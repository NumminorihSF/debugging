Debugging

Debugging is a process of finding and resolving of defects or problem within the program that prevent correct operation

Code-ways

  • alert();
  • console.log();
  • debugger;
  • monkey-patching
  • window[name] = anything;

alert();

Example

		function(arg) {
		  alert(arg);
		}

alert(); +

Example

  • Works everywhere
  • Does not require devtools
  • Consistent
  • Stops eventloop

alert(); -

Example

  • Stops eventloop
  • Waits user's iteraction

console.log();

Example

		function(arg) {
		  console.log(arg);
		}

console.log(); +

Example

  • Does not stop eventloop
  • Allows to see functions/links and other
  • User will not see (usually)

console.log(); -

Example

  • Non-consistent sometimes
  • Many logs are difficult to read

debugger;

Example

		function(arg) {
		  debugger;
		}

debugger; +

Example

  • Does not affect application without opened devtools
  • Can be ignored
  • Easy usage in case of broken source-maps
  • Write code and stops were you need
  • Allows to execute everything on pause

debugger; -

Example

  • Source-maps breaks debugging a bit
  • Good ways to inject own code

monkey-patching

Example

		setImmediate = function setIm(cb, ...args) {
		  return cb.call(this, ...args)
		}

monkey-patching +

  • Allows to do everything with source code

monkey-patching -

Example

  • Allows to do everything with source code
  • Difficult to use without breaking code

global variables

Example

		function (arg) {
		  window.a = arg;
		}

global variables +

  • Simple way to check statements anytime
  • Simple way to call some hidden methods to check effect

global variables -

  • Naming is a problem

Browser-ways

  • Dev tools
  • Extensions (redux-dev-tools, react-dev-tools and so one)

Dev tools

  • console
  • breakpoints
  • expression watcher
  • call stack explorer
  • scope explorer

Console

  • errors and theirs messages
  • store anything from current scope
  • execute the code

Breakpoints

  • pause on exception
  • conditional breakpoints
  • XHR breakpoints
  • DOM breakpoints
  • Event breakpoints

Breakpoint

Example

		function(arg) {
		  return arg;
		}

Breakpoint +

  • Easy to be managed from browser
  • Allows to see current scope as browser see it
  • Displays stacktrace and allow to traver over it
  • Allows to execute everything

Breakpoint -

  • May be annoying
